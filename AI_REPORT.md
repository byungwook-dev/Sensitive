# TeamBuilder AI - AI 기술 리포트

## 1. 프로젝트 개요

**TeamBuilder AI**는 교육기관(학교, 학원, 대학교)에서 팀/반 배정을 자동화하는 AI 기반 관리 시스템이다.
기존의 수작업 또는 랜덤 배정 방식의 비효율을 해결하고, 학생 데이터를 종합적으로 분석하여
성적, 성격, 성향, 성별, 나이 등 다양한 요소의 균형을 최적화한 배정 결과를 제공한다.

- **개발 기간**: 2026.04.08 ~ 진행 중
- **기술 스택**: Next.js 16 (App Router), TypeScript, Tailwind CSS, dnd-kit, Zustand, Claude API
- **AI 모델**: Claude Sonnet 4 (Anthropic)

---

## 2. AI 활용 영역

### 2.1 AI 자동 배정 엔진

**목적**: 학생들을 팀/반에 가장 균형 잡히게 자동 배정

**알고리즘 구조 (6단계 하이브리드 최적화)**:

| 단계 | 방식 | 설명 |
|------|------|------|
| 1단계 | 다중 전략 초기 배정 | 성적 스네이크, 성별 교차, 성격 라운드로빈, 성향 라운드로빈 4가지 전략 중 최선 선택 |
| 2단계 | 성적 집중 스왑 | 모든 팀 쌍에 대해 평균 성적을 전체 평균에 수렴시키는 교환 |
| 3단계 | 전체 균형 스왑 | 성격/성향/성별 등 전체 점수를 높이는 교환 (성적 보호 조건 포함) |
| 4단계 | 시뮬레이티드 어닐링 | 로컬 최적에서 탈출하여 글로벌 최적 탐색 (1500회 반복) |
| 5단계 | 성적 재균형 | 최종 성적 편차 재조정 |
| 6단계 | 최종 마무리 스왑 | 성적 보호하며 나머지 요소 미세 조정 |

**균형 점수 가중치**:

| 요소 | 가중치 | 설명 |
|------|--------|------|
| 성적 균형 | 30% | 팀 간 평균 성적 편차 최소화 |
| 성격 균형 | 26% | 6가지 업무 성격 유형 균등 분배 |
| 성향 균형 | 26% | 8가지 성향 유형 균등 분배 |
| 성별 균형 | 10% | 남녀 비율 전체 비율 대비 편차 |
| 나이 균형 | 4% | 팀 간 평균 나이 편차 |
| 인원 균형 | 4% | 팀 간 인원 수 편차 |

**성능 (33명 학생 기준)**:

| 팀 수 | 종합 점수 | 성적 | 성격 | 성향 | 평균 차이 |
|--------|----------|------|------|------|----------|
| 3팀 | 95.4 | 100 | 95.0 | 93.2 | 0.1점 |
| 4팀 | 91.3 | 98.7 | 87.2 | 87.7 | 0.5점 |
| 5팀 | 89.0 | 99.6 | 89.0 | 84.9 | 0.6점 |
| 6팀 | 81.3 | 97.1 | 88.9 | 63.3 | 0.6점 |

**성격 쏠림 방지**: 성적 스왑 시 한 팀에 같은 성격이 2명 이상 되는 교환은 자동 거부

**Claude API 연동**: 로컬 엔진과 Claude API 결과를 모두 생성한 후 균형 점수가 더 높은 쪽을 자동 선택

### 2.2 AI 데이터 정리 (Upload Cleanup)

**목적**: 비정형 Excel/CSV 데이터를 표준화된 학생 데이터로 변환

**처리 과정**:
1. Excel/CSV 파일 파싱 (xlsx 라이브러리)
2. Claude API에 원본 데이터 전송
3. AI가 수행하는 작업:
   - 컬럼명 자동 매핑 (학생명/성명/이름 → name)
   - 값 표준화 (남자/M/male → 남)
   - 자유 텍스트에서 성격 유형 추론 ("활발하고 친구들을 잘 이끈다" → 리더형)
   - 자유 텍스트에서 성향 추론 ("조용하고 꼼꼼한 편" → 내향적)
   - 누락/애매한 값 경고 표시
4. API 실패 시 로컬 키워드 매칭 엔진으로 자동 폴백

**지원 컬럼 변형**: 이름/성명/학생명, 점수/성적/총점, 남/남자/M/male 등 다양한 표기 자동 인식

---

### 2.3 AI 도우미 (재최적화)

**목적**: 관리자의 자연어 명령에 따라 기존 배정을 조정

**지원 명령 유형**:

| 명령 예시 | AI 동작 |
|----------|---------|
| "성적 균형 맞춰줘" | 평균 성적 가장 높은/낮은 팀 간 집중 교환 |
| "성격 유형 골고루" | 같은 성격 몰린 팀에서 분산 교환 |
| "성향 균형 맞춰줘" | 성향 기준 스왑 최적화 |
| "성별 비율 균등하게" | 남녀 비율 벗어난 팀에서 이성 교환 |
| "리더형 각 팀 1명씩" | 리더 없는 팀에 리더형 이동 (점수 유사 학생과 교환) |

**변경 전/후 비교**: 실행 전 균형 점수 비교 카드를 보여주고, 관리자가 확인 후 적용/취소 선택

---

### 2.4 AI 챗봇 (배정 현황 질의응답)

**목적**: 배정 현황에 대한 자유 질문에 답변

**제공 정보**:
- 학생 전체 데이터 (이름, 성별, 나이, 성격, 성향, 성적, 특이사항)
- 팀 배정 현황 (팀별 멤버 구성)

**질문 예시**:
- "평균 성적이 가장 높은 팀은?"
- "리더형이 없는 팀이 있어?"
- "성별 비율이 가장 안 맞는 반은?"
- "특이사항이 있는 학생은 누구야?"

---

### 2.5 AI 팀 분석 리포트

**목적**: 각 팀의 구성을 종합 분석하여 강점/약점/추천 활동 제시

**분석 항목**:

| 항목 | 내용 |
|------|------|
| 시너지 | 성격+성향 조합이 만드는 시너지 효과 |
| 성향 밸런스 | 외향적/내향적 비율 분석 |
| 성적 인사이트 | 성적 분포 특징과 학습 관점 조언 |
| 구성 분석 | 성별/나이 관점 분석 |
| 주의사항 | 잠재적 충돌 요인 (리더형 2명 등) |
| 추천 활동 | 이 팀에 적합한 수업/활동 추천 |
| 한줄 요약 | 팀 별명 (예: "균형잡힌 실행력 팀") |

**특이사항 반영**: 장애, 질환, 언어 문제 등 특이사항이 있는 학생이 팀에서 적절한 도움을 받을 수 있는지 분석

---

### 2.6 특이사항 기반 배려 배정

**목적**: 학생의 특이사항(자유 텍스트)을 AI가 이해하고 배정에 반영

**동작 방식**:
- 관리자가 특이사항에 어떤 내용이든 자유롭게 작성
- AI가 맥락을 파악하여 적합한 팀원과 매칭

**매칭 예시**:

| 특이사항 | AI 판단 | 매칭 대상 |
|----------|---------|----------|
| 지체장애 | 도움이 필요한 학생 | 협동형/배려심 있는 학생 |
| ADHD | 차분한 환경 필요 | 신중형/차분한 학생 |
| 영어 못함 | 언어 도움 필요 | 영어 잘하는 학생 |
| 소심/왕따 경험 | 사회적 지지 필요 | 사교적/외향적 학생 |
| 유학생 | 적응 도움 필요 | 배려심 있는 학생 |

## 4. AI 안전 설계

| 항목 | 설계 |
|------|------|
| API 실패 대응 | 모든 AI 기능에 로컬 폴백 엔진 내장 |
| 결과 검증 | 로컬 엔진과 AI 결과 중 균형 점수 높은 쪽 자동 선택 |
| 사용자 확인 | AI 도우미 변경 시 Before/After 비교 후 적용/취소 선택 |
| API 키 보안 | .env.local에 저장, 서버 사이드에서만 접근, .gitignore에 포함 |
| 데이터 보호 | localStorage에 저장, 외부 전송 없음 (API 호출 시에만 임시 전송) |

## 5. 향후 발전 방향

1. **데이터베이스 연동** - PostgreSQL/Supabase로 서버 영속 저장
2. **사용자 인증** - 관리자 로그인/권한 관리
3. **학생 그룹 고도화** - 그룹 간 학생 이동, 히스토리 관리
4. **배정 히스토리** - 이전 배정 결과 비교/복원
5. **AI 모델 업그레이드** - Claude Opus 4.6 적용으로 더 정교한 분석
6. **모바일 대응** - 반응형 레이아웃 최적화
7. **실시간 협업** - 여러 관리자가 동시 편집
8. **리포트 PDF 내보내기** - 팀 분석 결과를 PDF로 출력

## 6. 결론

TeamBuilder AI는 단순 랜덤 배정이 아닌, **AI가 학생의 성적·성격·성향·특이사항을 종합 분석하여
가장 합리적인 팀 구성을 자동으로 설계하는 시스템**이다.

6단계 하이브리드 최적화 알고리즘과 Claude API의 자연어 이해 능력을 결합하여,
관리자는 버튼 하나로 균형 잡힌 배정을 받고, 자연어로 세부 조정까지 할 수 있다.

특히 장애·질환·언어 문제 등 **특이사항을 가진 학생을 AI가 자동으로 배려하여 배정**하는 기능은
기존 배정 도구에서 찾기 어려운 차별화된 가치를 제공한다.
